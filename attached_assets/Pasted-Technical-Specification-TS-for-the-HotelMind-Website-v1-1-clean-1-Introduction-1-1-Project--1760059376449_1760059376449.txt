Technical Specification (TS) for the HotelMind Website (v1.1 — clean)

1. Introduction
1.1. Project: HotelMind corporate website — a platform of solutions for hotels, including the AI assistant Roomie.
1.2. Site goal: explain the value of HotelMind and Roomie for hotels, generate leads (demo/consultation), showcase case studies and benefits, build a mailing list for follow-ups and sales.
1.3. Target audience: owners and top managers of hotels/hostels/apart-hotels, front-office heads, marketers, COOs, chain managers (Europe and CIS). Digital maturity: from basic to advanced.

⸻
2. Business Objectives and KPIs
2.1. Objectives:
 • Increase trust in the HotelMind brand and the Roomie product.
 • Launch a steady lead flow (forms, demo request, chat widget).
 • Demonstrate durable benefits: growth of direct bookings, upsells, reduced staff workload, faster response time.
2.2. KPIs:
 • Session → request/demo conversion (target set during analytics).
 • Time-to-first-interaction (TTI) inside the Roomie chat.
 • Share of traffic that viewed “Benefits” / “Case Studies”.
 • Core Web Vitals: LCP, INP, CLS — see section 8.

⸻
3. Brand and Visual Style
3.1. Palette: a single accent color HotelMind Blue #0752A0 (from the logo) + its shades and neutrals (black/graphite/gray/white). No other accent colors.
3.2. Style: contemporary premium minimalism (think “expensive studios”): generous whitespace, strict grid, large typography, subtle micro-animations.
3.3. Fixed palette:
 • Accent: #0752A0.
 • Shades: #4778A8, #5989B6, #709DC4, #92B4CD, #ACCAE0.
 • Neutrals: Charcoal #111111; Graphite #1F1F1F; Grey #8A8A8A; Off-white #F6F6F4; White #FFFFFF.
3.4. Typography:
 • Headings: Fraunces / Playfair / Cormorant (one), optionally Manrope.
 • UI text: Inter or Manrope. Body 16–18 px, line-height 1.5×.
3.5. UI components: soft-shadow cards, large CTAs, line icons, abstract illustrations + Roomie UI mockups.
3.6. Imagery: real hotel photos (lobby/reception/rooms), Roomie chat screenshots/mockups, ROI charts.
3.7. Animations: fade/slide, light hero parallax, benefit counters, “assistant is typing” indicator.

Breadcrumbs and structured data — see section 9.

5. Content Skeleton of Key Pages
5.1. Home:
 • Hero: USP + short subtitle + primary CTA “Request a demo” + secondary CTA “See case studies”. Background: static or light video loop (no blue filter).
 • “How Roomie works”: 3–5 steps (guest question → understanding → personalized answer → upsell → action/booking).
 • Integrations (ticker): logos of Telegram, PMS Servio, Tilda, WordPress, Wix, website icon (widget), WhatsApp (“Coming soon”), Instagram (“Coming soon”). Logos are monochrome/outline, with no blue/cyan tones (see 7.10).
 • Benefits (see 6.1) + supporting numbers.
 • Client logos/case tiles.
 • CTA “Get ROI estimate” (mini-form).
 • Important: there is NO embedded demo chat on Home; access to demo only via the header button (see 7.11).
5.2. Roomie (product):
 • Modules: multilingual dialog, “shadow guest card”, personalization & upsells, FAQ answers, analytics dashboard (FAQ clicks, query frequencies), segmented campaigns, integrations with PMS/channel manager/messengers, website widget.
 • Architecture diagram (clear for non-tech readers).
 • Security & privacy.
 • Simple on-page ROI calculator.
5.3. Solutions: three subpages with differences in processes, scalability, channels, integrations.
5.4. Case Studies & Testimonials: “problem → solution → result (numbers)” cards, videos/quotes.
5.5. Pricing: plan table + “What’s included”, channel/volume limits, payment FAQ, CTA “Request a custom quote”.
5.6. Blog: articles on personalization, seamless journeys, integrations, RevPAR growth.
5.7. Contacts: form (name, role, property, phone, email, comment), demo calendar, map, legal info.

⸻
6. Value Messaging (copy guidelines)
6.1. Key benefits:
 • More direct bookings (site/messenger dialog, objection handling).
 • Lower reception workload: AI covers routine 24/7.
 • Personalized upsells: late checkout, room upgrade, breakfast/SPA, transfer — based on behavior signals.
 • Multilingual support without extra staff.
 • “Shadow guest card”: preferences/history/UTMs for personalization.
 • SLA measured in seconds, higher NPS/CSAT.
 • Transparent analytics: questions, FAQ clicks, conversions, lead sources.
 • Fast integrations: PMS/CRM/channel manager, messengers, site widget.
6.2. Tone: confident and businesslike, no jargon, short paragraphs, concrete benefits.

⸻
7. Functional Requirements
7.1. Forms:
 • All forms (lead, contact, ROI) integrate with MailChimp: create/update contact, Tags, Merge fields.
 • Double opt-in: ON by default (toggleable).
 • Fields: name, role, property, phone, email, comment, separate checkboxes for data processing and marketing emails; hidden UTM/referrer/language.
 • Anti-spam: hCaptcha.
 • “Thank you” (page/modal) + MailChimp system email.
7.2. Roomie demo:
 • On the site — description only; demo access via the header button (see 7.11).
 • For the demo domain: a top-right slot for 2–4 buttons (label+icon) linking to specified domains; #link-1/2… placeholders at dev stage; configured via .env/CMS (text, URL, order, visibility), multilingual labels.
7.3. Integrations:
 • MailChimp REST API (Tags/Merge fields incl. UTM/source), double opt-in.
 • No external analytics/CRM; basic metrics can be collected via server logs/hosting if needed.
7.4. Site search: typeahead + results with snippets.
7.5. Multilingual: RU/UA/EN; auto-detected by browser language + manual header switcher; i18n-friendly URLs and hreflang.
7.6. Blog: categories, tags, authors, search, scheduled publishing, preview.
7.7. SEO management: title, description, OG, canonical, schema, redirects, sitemap, robots.txt.
7.8. Media management: crop/optimization, WebP/AVIF, lazy-loading, auto-generated social previews.
7.10. Integrations slot on Home (ticker):
 • Logos: Telegram, PMS Servio, Tilda, WordPress, Wix, website icon (widget), WhatsApp, Instagram.
 • Badges: WhatsApp/Instagram carry a subtle “Coming soon” micro-badge (low emphasis) with aria-label="Coming soon" and tooltip on focus/hover.
 • Style: monochrome/outline, strictly no blue/cyan (to avoid competing with brand accent).
 • Mechanics: CSS loop (no <marquee>), 30–45 s cycle; pause on hover/focus. On mobile — swipe/horizontal scroll, no auto-scroll.
 • Performance: lazy-loading, respects prefers-reduced-motion, zero CLS (fixed container height), init via IntersectionObserver.
 • Accessibility: role="list"/"listitem", alt/aria-label per logo.
 • Management: composition/order/links/badges configurable in CMS; default links — #integration-… .
7.11. Global header and CTA “Try the demo”:
 • Button in the top-right of the header, linking to an external demo domain (provided by the Client).
 • A plain anchor link; no embedded SDK/iframe of the demo on HotelMind.
 • In CMS: localized label (RU/UA/EN), URL, target (same/new tab), icon on/off, breakpoint visibility, order.
 • States: hover/focus/disabled/active; light/dark; contrast ≥ 4.5:1.
 • Tracking: local click counter (aggregated), optional webhook notifications.

⸻
8. Non-Functional Requirements
8.1. Core Web Vitals (mobile 4G): LCP ≤ 2.5 s, CLS ≤ 0.1, INP ≤ 200 ms. Critical CSS+JS before interactivity ≤ 180 KB (guideline).
8.2. Responsive: mobile-first; breakpoints 360/768/1024/1280/1536 px.
8.3. Accessibility: semantic HTML, ARIA, contrast ≥ 4.5:1, visible focus, keyboard support, alt texts.
8.4. Reliability: no critical Lighthouse/axe issues; graceful degradation without JS for main text blocks.

⸻
9. SEO and Structured Data
 • Schema.org: Organization, Product (Roomie), FAQPage, Article (blog), BreadcrumbList, WebSite/SearchAction (search).
 • Human-readable URLs, sitemap XML, robots.txt, hreflang for RU/UA/EN.
 • Open Graph/Twitter Cards; preview images with blue dominance and sufficient contrast.

⸻
10. Technical Architecture and Stack
 • Frontend: Next.js (React, TypeScript), Tailwind CSS, shadcn/ui.
 • Headless CMS: Sanity/Strapi/Contentful (final choice at initiation).
 • Hosting/deploy: Vercel/analog with CDN, edge caching, image optimization.
 • Forms: server endpoints → MailChimp API (Audience, Tags, Merge fields) + hCaptcha.
 • Logs/monitoring: Sentry (errors), basic uptime ping.

⸻
11. Admin Panel (Studio): structure, roles, management
Implementation: headless CMS (Sanity Studio / Strapi) as a separate /admin app with roles, preview, and audit log.

11.1. Menu:
 • Dashboard: publishing statuses, missing required fields, localization progress, accessibility warnings, brand control (color allowlist), quick links.
 • Pages: Home, Roomie, Solutions, Case Studies, Pricing, Blog, About, Contacts, Roomie Demo, 404/5xx.
 • Components/Sections: Hero, Benefits, Integrations (ticker), Case feed, CTA block, Pricing table, FAQ, Nav/Menu, Footer.
 • Content: Blog posts (categories/tags), Case studies, Testimonials.
 • Media library: images/logos/video/SVG (incl. monochrome integration logos), focal point, WebP/AVIF generation.
 • Settings: domains, locales/hreflang, default SEO (title/description templates), OG templates, redirects (301/308), robots.txt, sitemap.
 • Roomie Demo: widget config (see 11.7), scenarios, feature flags.
 • Integrations: MailChimp, integrations ticker editor, webhook notifications.
 • Legal: Privacy Policy, Terms, Cookie Policy.
 • Access & audit: users/roles, activity log, tokens, webhooks.
 • Support: form submission logs (MailChimp responses), Sentry errors, CDN cache purge.
11.2. Roles:
 • Admin — full access, production publish, role management.
 • Editor — content and publish (no system settings).
 • Marketer — sections, pricing, CTAs, blog, ticker, MailChimp tags.
 • Legal — legal pages and consent texts.
 • Developer — site settings, scripts/meta, redirects.
 • Translator — localizes strings without changing structure.
11.3. Locales/management:
 • Modes: Draft / Publish / Schedule (by time and time zone).
 • Preview via protected link.
 • Localization: RU/UA/EN with statuses “to translate / up-to-date / outdated”, default fallback.
 • Validation: title/meta length limits, required alt texts, unique slugs, no hreflang duplicates.
11.4. SEO/OG settings:
 • Per page: title, description, canonical, indexability, OG title/description, OG image (server-side cover generator), JSON-LD.
 • Global: robots.txt, sitemap.xml regeneration, 301/308 redirect manager (collision checks).
11.5. “Integrations (ticker)” editor:
 • IntegrationItem collection: platform (enum), link, icon (SVG/PNG), alt/aria-label, “Coming soon” toggle, order (drag-and-drop), visibility (by locale/env).
 • Auto-checks: allowed colors (monochrome, no blues), minimum sizes, “Coming soon” badge contrast.
 • Mobile preview for carousel/swipe.
11.6. Forms & MailChimp:
 • Settings: Audience ID, API key (env), double opt-in (toggle), field mapping (name, role, property, phone, email, comment, UTM, language) → Merge fields, Tag rules.
 • Test send: “Send test” (shows payload+status), retry queue on failures.
 • Logs: recent submits (subscribed/pending/cleaned), CSV export.
11.7. Roomie demo — configuration:
 • Top-right buttons: DemoLink collection (label RU/UA/EN, icon, URL, order, visibility). URLs may be empty until domains are provided.
 • Widget themes: presets in brand blue, contrast control.
 • Script/iframe embed: code generation, in-studio preview, “disable in production” checkbox.
 • Disclaimers and demo scenarios (read-only mode, phrase limitations).
11.8. Brand guardrails:
 • Color allowlist: accent #0752A0 and neutrals (black/white/off-white/graphite). Any other accents are forbidden.
 • Contrast WCAG 2.1 AA ≥ 4.5:1 for text/CTAs/icons.
 • Typography: fonts/sizes/spacing; icon/button library; unified style guide.
11.9. QA & accessibility:
 • Pre-publish checklist: alts, sizes, contrast, translations, broken links, CLS-safe blocks.
 • (Optional) Lighthouse run in preview (targets in 8.1).
11.10. Releases & versions:
 • Staging/Production, content freeze, Purge CDN.
 • Change history, diff & rollback.
 • JSON/CSV import/export (translations, cases, pricing).
11.11. Notifications:
 • Webhooks: “new lead”, “form error”, “page published” → email/Slack.
11.12. Security & privacy:
 • Roles/access, 2FA for admins, activity audit.
 • Secrets via env vars; upload size limits.
 • Lead data stored in MailChimp; only submit metadata in admin.
11.13. System utilities:
 • Generate/regenerate sitemap, check hreflang, find orphan pages.
 • Link checker (404/redirects), canonical duplicates finder.
11.14. Cookie Policy consent log (CMP):
 • Events: accepted/declined/changed/withdrew; timestamp (UTC), locale, policy version, categories set, page, source (banner/panel), status (active/expired), session/client ID.
 • Identification: pseudonymous consent_id (UUID) in a first-party cookie; IP anonymized (IPv4 /24, IPv6 /56); user-agent hashed.
 • Filters: by date, locale, version, region, event, category.
 • Export: CSV/JSON for a period; aggregated stats (category consent shares, declines, changes).
 • Management: policy versioning, auto re-consent, emergency revoke-all button.
 • Linked items: Cookie Policy editor (RU/UA/EN) and banner settings (texts, categories, default behavior, optional IAB compatibility).

⸻
12. Legal and Compliance
 • Privacy Policy, Terms of Use, cookie consent banner.
 • Compliance with GDPR and local Ukrainian PD laws; consider ePrivacy/PECR.
 • Email marketing: explicit checkboxes (data processing; marketing emails), MailChimp double opt-in, mandatory Unsubscribe.
 • Lead personal data stored in MailChimp (data minimization); role-based access.

⸻
13. Content: volume and responsibility
 • Texts: USP (up to 120 words), benefits (6–8), case studies (3–6), solutions (3), Roomie product (full module description), pricing (table + FAQ), initial blog 5–10 posts.
 • Media: 10–15 photos, 4–6 Roomie UI mockups, 2–3 ROI diagrams.
 • Languages: RU/UA/EN — parity across key pages.

⸻
14. Lead Tracking (no Google/CRM)
 • Lead accounting is based on a contact entering MailChimp Audience (subscribed/pending/cleaned) with Tags/Merge fields for source/UTM/language/page.
 • Reports export: CSV/Excel from MailChimp; if needed — MailChimp webhooks to an internal endpoint (email/Slack).

⸻
15. Roomie Integrations and Demo
 • Demo is hosted on a separate domain/subdomain provided by the Client. The HotelMind site only contains the link/button “Try the demo” (see 7.11) and a description page — without embedded chat.
 • On the demo domain: 2–4 configurable buttons (label+icon); order/links set by the Client; placeholders by default.
 • Demo scenario (FAQ, upsell, booking request): a limited phrase set for a safe experience.

⸻
16. Design Deliverables
 • Figma: design system (colors, typography, grids, components), layouts for 3 breakpoints (mobile/tablet/desktop), interactive prototypes.
 • Components: buttons, cards, pricing tables, popups, tabs, accordions, forms, charts, integrations ticker (with “Coming soon” badges for WhatsApp/Instagram).
 • Asset export 1×/2×, SVG for icons.

⸻
17. Testing & Acceptance
 • Autotests for critical paths (forms, language switcher, routing).
 • Lighthouse (mobile/desktop): Performance/Accessibility/SEO ≥ targets (see 8.1).
 • Cross-browser: Chrome/Safari/Firefox/Edge (last 2 versions), iOS/Android.
 • Integration checks: MailChimp (double opt-in, tags/merge), webhook notifications, demo button/redirect.

Acceptance criteria (must-have):
1) Single palette: HotelMind Blue #0752A0 + neutrals only; contrast meets WCAG 2.1 AA.
2) Multilingual RU/UA/EN launched, correct hreflang.
3) No embedded demo chat on Home.
4) Header has the “Try the demo” button on the right; label/URL/target are configurable in CMS and link to the demo domain.
5) Forms send data to MailChimp Audience (with Tags and Merge fields); double opt-in enabled (unless stated otherwise); anti-spam works.
6) Integrations slot implemented: ticker/carousel with logos Telegram, PMS Servio, Tilda, WordPress, Wix, website icon, WhatsApp (“Coming soon”), Instagram (“Coming soon”); accessibility & responsiveness as specified.
7) Cookie Policy implemented (RU/UA/EN), banner and management panel with categories and EU opt-in geo-mode. Footer has a “Change cookie settings” page.
8) Consent log (CMP) available in admin: events, filters, CSV/JSON export, policy versioning and re-consent.
9) Core Web Vitals in the “green zone” (see 8.1).
